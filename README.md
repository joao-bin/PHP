## Sobre

Uma linguagem de programação open source de uso geral, especialmente utilizada em desenvolvimento web e que pode ser mesclada com HTML. É o PHP que faz diversos sites que você conhece e usa diariamente funcionarem, como Slack, Facebook e WordPress.

# História

A primeira versão do PHP foi lançada em 1994 por Rasmus Lerdorf, que inicialmente tinha como objetivo criar scripts simples para monitorar os acessos do seu website. Feita na linguagem C, na época se chamava "Personal Home Page" ou "PHP Tools". Ao longo do tempo foi sendo aprimorada com novas funcionalidades, como conexão com banco de dados e geração de páginas de forma dinâmica. Em 1995, Rasmus liberou o código-fonte do PHP Tools para que outros desenvolvedores pudessem contribuir.

O PHP Tools passou por expansões, reescritas, mudanças de sintaxe e de nome. Uma dessas versões reescritas foi o PHP/FI, que ganhou popularidade e em 1998 já estava presente em cerca de 60.000 domínios.

Em 1997, os universitários Andi Gutmans e Zeev Suraski precisavam desenvolver uma aplicação de e-commerce e perceberam que o PHP/FI não possuía alguns recursos de que precisariam. Ao contatarem Rasmus e discutirem sobre as funcionalidades da ferramenta, decidiram juntos reescrevê-la como uma nova e independente linguagem de programação.
Assim nasce a linguagem PHP, um acrônimo recursivo de "Hypertext Preprocessor".

# As principais características do PHP

- **Linguagem Server-Side**: O código é processado no servidor web antes de chegar ao navegador do usuário, o que garante segurança ao esconder a lógica da aplicação.

- **PHP embutido com HTML**: Scripts de PHP podem ser mesclados com HTML, facilitando a criação de páginas web.

- **Alta compatibilidade com Banco de Dados**: Oferece suporte nativo e excelente integração com uma vasta gama de bancos de dados, como MySQL, PostgreSQL, SQLite e Oracle.

- **Paradigma de Programação**: Suporta múltiplos paradigmas de programação, permitindo desde códigos estruturados simples até o uso avançado de Programação Orientada a Objetos.

## O que pode ser feito no PHP?

O PHP é uma linguagem versátil, capaz de se adaptar a diversas áreas, sendo as mais utilizadas as seguintes:

- **Desenvolvimento de Websites**: o uso mais comum do PHP. Ele foi criado para gerar páginas dinâmicas na web, processando dados no servidor e interagindo com bancos de dados.

- **Aplicações em linha de comando**: é possível utilizar o PHP para execução de scripts de linha de comando totalmente independentes do servidor web. Isso permite a manipulação de arquivos e a realização de tarefas manuais diretamente pelo terminal do seu computador.

- **APIs e serviços web**: o PHP é uma excelente opção para a construção de APIs RESTful e serviços web. Com ele, é possível criar endpoints que recebem e retornam dados em formatos como JSON e XML, facilitando a integração entre sistemas e aplicações distintas. Frameworks como Laravel tornam esse processo ainda mais ágil, oferecendo recursos nativos para roteamento, autenticação e formatação de respostas.




# Principais Frameworks utilizados para Desenvolvimento Web com PHP
## Laravel  <img width="50" height="50" alt="image" src="https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcTCFtyvyHW25S9KRpXy9NfNsNT3JKyF1vmUPccNADeavg&s=10">
O Laravel é reconhecido mundialmente pelo seu rápido crescimento em popularidade. É um framework PHP acessível e robusto, que entrega as ferramentas necessárias para o desenvolvimento de qualquer tipo de aplicativo.
### Características principais
- **Sintaxe elegante e intuitiva**: Laravel é conhecido por sua curva de aprendizado suave. Sua sintaxe foi projetada para ser clara e simples, o que facilita a escrita e a leitura do código.

- **Sistema de rotas eficaz**: oferece um sistema de roteamento que permite mapear URLs para controladores e funções de forma simples, mas eficiente. Essa flexibilidade é muito útil na criação de APIs e na organização dos endpoints.

- **Eloquent ORM**: o Eloquent é uma ferramenta de mapeamento objeto-relacional que permite manipular dados de maneira intuitiva, usando uma sintaxe que facilita o relacionamento entre tabelas e a construção de consultas complexas.

## CakePHP <img widht= "50" height = "50" alt="image" src = "https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcR_WM4gIKYnLzRg3mOe-P5rpN4ZmAsRxG3-4QZAyDUDbQ&s=10">

Lançado em 2005, o CakePHP ganhou destaque por adotar o princípio "convenção sobre configuração". Essa abordagem diminui o tempo necessário para a configuração inicial, permitindo que o desenvolvedor foque diretamente na lógica da aplicação.

- **Convenção sobre configuração**: o CakePHP impõe um conjunto consistente de convenções (como nomenclatura de arquivos e estrutura de pastas) que facilitam o desenvolvimento e a padronização das aplicações.

- **Scaffolding**: as ferramentas integradas permitem a geração automática de código e CRUDs (operações de criação, leitura, atualização e remoção), acelerando o desenvolvimento inicial e a prototipagem.

- **Integração com banco de dados**: com recursos avançados para manipulação de dados e mapeamento objeto-relacional, o CakePHP facilita a interação com diversos tipos de banco de dados.



# PRÓS E CONTRAS DO PHP


## PRÓS
 
 - Possui uma linguagem acessível e amigável, o que a torna uma boa opção para iniciantes.
 
 - Multiplataforma, ou seja, roda em todos os sistemas operacionais, como Windows, Mac e Linux.

 - Grande oferta de trabalho: devido à enorme quantidade de sites na internet que utilizam PHP, há uma demanda constante por desenvolvedores.
 
 - As versões mais recentes do PHP trouxeram ganhos massivos de velocidade e otimização de memória.

## CONTRAS

- A linguagem tem um longo histórico, o que gera inconsistências nos nomes de funções e APIs, além de muitos códigos mal estruturados rodando por aí.

- Por padrão, a tipagem é fraca, o que pode abrir portas para bugs sutis caso o desenvolvedor não seja disciplinado.

- Menos versátil fora da web: embora existam maneiras de contornar, o PHP não é a primeira opção para criar aplicativos mobile ou softwares de desktop.

- Comparado a linguagens projetadas para processamento assíncrono, o modelo tradicional do PHP lida pior com operações de altíssima concorrência e tempo real.
